Die drei Rollen im System

Admin (du)

Legt alles an. Startet Jobs. Kontrolliert den gesamten Ablauf. Zugang: /pb-manage/

Studio Worker

Erstellt Jobs (Draft), sichtet Ergebnisse, selektiert Bilder, freigibt für Shop/Galerie. Kein Job-Start.

FulfillmentPartner

Druckstudio. Sieht offene Aufträge, lädt Bundle herunter, trägt Tracking-Code ein.

Schritt 2: Job anlegen

Admin → Jobs → Jobs → + Job hinzufügen

  1. Titel

    Interner Name, z.B. "Poster Herbstwald 01" — erscheint nicht öffentlich

  2. Pipeline Template wählen

    Aus Dropdown: "Poster Production" (oder was du in Schritt 1 angelegt hast)

  3. Prompt schreiben

    Was beschreibt das Bild präzise? Stil, Motiv, Farben, Lichtstimmung — je konkreter desto besser.

    A misty autumn forest at golden hour, warm orange and yellow foliage,
rays of light through tall pine trees, photorealistic, 8K, cinematic lighting,
shallow depth of field, highly detailed, nature photography style
  4. Negative Prompt

    Was soll das Modell vermeiden.

    blurry, oversaturated, cartoon, anime, low quality, watermark, text,
people, buildings, artifacts, noise, distorted
  5. Parameter-Overrides (optional)

    Leer lassen → Template-Defaults werden verwendet. Nur überschreiben wenn nötig:

Parameter-Referenz — Was ändert was?

ParameterTypEffektTypische Werte
stepsintAnzahl Diffusions-Schritte. Mehr = schärfer, mehr Details, aber langsamer. Ab ~40 kaum noch Unterschied.20 (schnell) · 30 (standard) · 50 (max)
guidancefloatWie stark der Prompt befolgt wird. Zu hoch = übergesättigte Farben, steife Komposition. Zu niedrig = Prompt wird ignoriert.5.0–12.0 · Standard: 7.5
seedintZufallszahl. Gleicher Seed + gleicher Prompt = identisches Bild. Für Varianten: Seed leicht ändern.leer = zufällig · z.B. 42 für Reproduzierbarkeit
width / heightintAusgabe-Auflösung vor Upscaling. 1024×1024 → nach Upscale 4096×4096 (~13MP).1024×1024 (quadrat) · 768×1152 (Hochformat)
num_imagesintWie viele Variationen parallel generiert werden. Jede zählt als ein Job-Output.1–4 (abhängig von RunPod-Budget)
modelchoiceDas KI-Modell. flux_schnell ist schnell + kommerziell. sdxl für anderen Stil. flux_dev NUR intern.flux_schnell (Prod) · sdxl · flux_dev (Test)

Schritt 3: Job starten (Admin-Aktion)

Studio-Worker können Jobs erstellen aber nicht starten (ADR-11). Nur Admins können Jobs starten.

  1. Job-Liste öffnen

    Admin → Jobs → Jobs

  2. Job(s) auswählen

    Checkbox links anklicken

  3. Aktion ausführen

    Dropdown "Aktion" → "Start selected jobs" → Ausführen

    Status wechselt von draft zu queued, dann running

  4. Fortschritt verfolgen

    Admin → Jobs → Job Steps (oder Studio-View /studio/job/<uuid>/ — HTMX-Polling alle 3s)

Schritt 4: Bild in der Datenbank finden

Nach Abschluss der Pipeline (status=done) liegt das Bild als JobStep-Output und als Datei vor.

Über Admin

Admin → Jobs → Job Steps → Filter: step_type=preview_export, status=done → output_asset_id notieren

Oder: Admin → Gallery → Gallery Images (wenn Studio-Worker das Bild bereits selektiert hat)

Django Shell (für Debugging)

# VPS: ssh root@167.86.108.37
# cd /opt/printbuddy && venv/bin/python manage.py shell

from jobs.models import Job, JobStep

# Letzten abgeschlossenen Job finden
job = Job.objects.filter(status='done').latest('completed_at')
print(job.id, job.title)

# Alle Steps eines Jobs ansehen
for step in job.steps.all().order_by('order'):
    print(f"{step.step_type:20} {step.status:8} {step.output_asset_id}")

# Preview-Datei-Pfad
preview = job.steps.get(step_type='preview_export', status='done')
print(preview.output_asset_id)  # UUID des Bildes

2 — Partner einpflegen: Vertrieb & Druck

Es gibt zwei getrennte Konzepte: SalesChannel (wo wird verkauft) und FulfillmentPartner (wer druckt).

SalesChannel — Vertriebskanal

Wo eine Bestellung herkommt oder hin gemeldet wird. Beispiele: eigener Shop, Etsy, WooCommerce-Shop eines Wiederverkäufers.

FulfillmentPartner — Druckstudio

Wer die Bestellung physisch produziert und versendet. Bekommt einen eigenen Login und sieht seine Aufträge im Partner-Dashboard.

4 — Mockup-Workflow

Mockups sind die "Lifestyle"-Bilder: dein Bild auf einem Poster-Rahmen, einem T-Shirt, etc. Sie kommen aus der Printful Mockup API oder werden manuell erstellt.

Option A: Printful Mockup API (automatisch)

Printful bietet für jedes ihrer Produkte eine Mockup-Generierung. Das System nutzt die API um für jedes ImageProduct ein Mockup zu erzeugen.

ImageProduct angelegt
→
generate_all_mockups Task
→
Printful API
→
Mockup-Bild URL
→
ImageProduct.mockup_file = ready

Printful Mockup API — wie sie funktioniert

# Schritt 1: Mockup-Task erstellen
POST https://api.printful.com/mockup-generator/create-task/{product_id}
{
  "variant_ids": [4011],           # Printful Variant ID
  "format": "jpg",
  "files": [{
    "placement": "default",
    "image_url": "https://printbuddy.datemyhobby.com/media/gallery/full/herbstwald.jpg",
    "position": {
      "area_width": 1800,
      "area_height": 2400,
      "width": 1800,
      "height": 2400,
      "top": 0,
      "left": 0
    }
  }]
}
# Response: {"task_key": "gt_abc123", "status": "pending"}

# Schritt 2: Ergebnis pollen bis status=completed
GET https://api.printful.com/mockup-generator/task?task_key=gt_abc123
# Response bei Erfolg: {"status": "completed", "mockups": [{"mockup_url": "https://..."}]}

# Schritt 3: Mockup-Bild herunterladen + in ImageProduct.mockup_file speichern

mockup_status — Lifecycle

StatusBedeutungWas passiert

Option B: Eigenes Mockup manuell hochladen

Für den MVP oder wenn kein Printful-Produkt passt: Mockup in Canva, Photoshop oder einem anderen Tool erstellen und direkt hochladen.

  1. Mockup erstellen

    Tool: Canva, Photoshop, Smartmockups.com — dein Bild auf einen Rahmen/Wand/Tisch platzieren

  2. Hochladen

    Admin → Shop → Image Products → Eintrag öffnen → mockup_file → Datei wählen → mockup_status auf ready setzen

  3. Im Shop

    Der ProductDetailView zeigt mockup_file wenn mockup_status=ready, sonst base_image als Fallback

Pro GalleryImage können mehrere ImageProduct-Einträge existieren — eines pro Produkttyp (Poster A4, Poster A3, Shirt). Jeder bekommt sein eigenes Mockup.

Wie das Bundle für den Partner gebaut wird

Nach einer bezahlten Bestellung läuft der create_print_bundle Task:

  1. OrderLines durchgehen

    Für jede OrderLine: ProductVariant.required_export_types bestimmen

  2. Asset-Dateien sammeln

    Die passenden Exporte für die required_export_types aus den JobSteps holen

  3. ZIP erstellen

    Alle Dateien in /mnt/agency_nas/bundles/<order_id>.zip packen

    bundle.zip/
  herbstwald_poster_a2_pod.png     # PNG 300dpi sRGB
  herbstwald_poster_a2_cmyk.tiff   # CMYK TIFF
  herbstwald_poster_a2_print.pdf   # PDF/X-4 mit Bleed
  herbstwald_poster_a2_preview.jpg # Vorschau
  order_info.json                  # Bestelldetails, Adresse, Tracking-Info
  4. Partner benachrichtigen

    notify_fulfillment_partner Task: E-Mail mit Bundle-Download-Link + optional Webhook an Partner-System

  5. Partner-Dashboard

    Partner loggt sich in /partner/ ein → sieht offene Aufträge → lädt Bundle herunter → trägt Tracking-Code ein

Required Export Types — wie sie gematched werden

ProductVariant.required_export_typesFulfillmentPartner.export_formatsMatch?
["pod","preview"]["pod","cmyk"]✓ Ja — pod vorhanden
["cmyk","preview"]["pod"]✗ Nein — cmyk fehlt beim Partner
["pod","cmyk","vector"]["pod","cmyk","vector"]✓ Ja — alle vorhanden

Das System wählt den Partner der alle required_export_types liefern kann. Wenn kein passender Partner → Fallback auf Printful (für pod) oder manuell (für cmyk/vector).

Formate im Detail

POD Export — PNG 300dpi sRGB

  • Format: PNG (verlustfrei)
  • Farbraum: sRGB (wichtig für Printful)
  • Auflösung: 300dpi
  • Größe: typisch 3600×3600px bei A4 300dpi
  • Kein Beschnitt nötig
  • Direkt an Printful API übergeben

CMYK Export — TIFF + PDF/X-4

  • Farbraum: CMYK (ISO Coated v2)
  • Format: TIFF (Rohdaten) + PDF/X-4 (druckfertig)
  • 3mm Beschnitt (Bleed) an allen Seiten
  • Auflösung: 300dpi
  • Ghostscript für PDF/X-4 Konvertierung
  • Für Offset-, Digital- und Großformatdruck

Vector Export — SVG

  • Inkscape CLI + Potrace
  • Skalierbar (Vektordaten)
  • Ideal für Grafiken, Logos, Icons
  • Nicht für fotografisch-realistische Bilder
  • Für Siebdruck, Plotter, Lasergravur

Preview Export — JPG 72dpi

  • JPG, maximal 1200px Breite
  • 72dpi (Web-Optimiert)
  • Kleine Dateigröße, schnelles Laden
  • Für Galerie, Shop, Social Media
  • Immer generiert, unabhängig vom Template

6 — Kompletter End-to-End Flow

Dieser Tab zeigt den gesamten Ablauf von der ersten Idee bis zum gedruckten Produkt beim Kunden.

Phase A: Setup (einmalig)

  1. PipelineTemplates anlegen

    Admin → Jobs → 2–3 Templates anlegen (POD Standard, Offset, evtl. Vektor)

  2. FulfillmentPartner anlegen

    User anlegen → Gruppe print_partners → FulfillmentPartner-Objekt mit export_formats

  3. SalesChannel anlegen

    Eigener Shop + ggf. Etsy-Kanal in Channels

  4. Produkt-Grundstruktur

    Products + Variants anlegen (ohne Bilder, noch kein Mockup)

Phase B: Bild-Produktion (pro Produkt)

  1. Job erstellen

    Admin (oder Studio Worker) → Jobs → + Hinzufügen → Prompt + Template wählen → Speichern (status=draft)

  2. Job starten

    Admin → Jobs-Liste → Job auswählen → "Start selected jobs" → status=queued → running

  3. Pipeline läuft automatisch 
    generate_image       # RunPod generiert 1024×1024
  ↓
upscale_image        # Real-ESRGAN → 4096×4096
  ↓
pod_export           # PNG 300dpi → /exports/pod/
  ↓ (optional)
cmyk_export          # TIFF + PDF/X-4 → /exports/offset/
  ↓
preview_export       # JPG 72dpi → /exports/preview/
  ↓
notify_studio        # Job.status = done
  4. Ergebnis sichten

    Studio Worker → /studio/job/<uuid>/results/ → Bilder ansehen → Beste auswählen

  5. Bild freigeben (Asset Select)

    Studio → "Dieses Bild freigeben" POST → legt GalleryImage + ImageProduct an → startet generate_all_mockups

  6. GalleryImage bearbeiten

    Admin → Gallery Images → Titel, Kategorie, cta_type, is_public=✓ setzen

Phase C: Produkt-Setup (pro Bild)

  1. Mockup abwarten oder manuell hochladen

    Wenn generate_all_mockups läuft: mockup_status wird → ready. Manuell: Admin → Image Products → mockup_file hochladen

  2. Produkt aktivieren

    Admin → Shop → Products → is_active = ✓ → Produkt erscheint in /shop/

  3. Etsy Listing erstellen (optional, Phase 8)

    Studio → Asset selektieren → "Als Etsy Listing erstellen" → etsy/listing.py erstellt Draft-Listing

Phase D: Bestellung & Fulfillment (pro Bestellung)

  1. Kunde kauft

    /shop/checkout/ → Stripe Hosted Checkout → Zahlung erfolgt

  2. Stripe Webhook

    POST /shop/webhook/stripe/ → construct_event() Signatur-Prüfung → Order.status = paid

  3. Automatische Tasks starten 
    create_print_bundle      # ZIP mit Druckdateien → /mnt/agency_nas/bundles/
create_printful_order    # (wenn Printful-Variante) → Printful API
notify_fulfillment_partner # E-Mail + Webhook an Partner
push_order_to_channel    # Bestellbestätigung an SalesChannel
  4. Partner produziert

    /partner/orders/ → Bundle herunterladen → drucken → /partner/order/<uuid>/shipped/ → Tracking-Code eintragen

  5. Kunde informiert

    Order.status = shipped → E-Mail mit Tracking an Kunden (via notify-Task)

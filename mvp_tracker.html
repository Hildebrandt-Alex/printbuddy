📖 Projektstruktur erklärt

Warum settings/ aufteilen?

Lokal läuft SQLite (schnell, kein Server nötig), auf dem VPS PostgreSQL 15. DEBUG=True lokal erlaubt, DEBUG=False auf VPS Pflicht. local.py importiert from .base import * und überschreibt nur was nötig ist.

Warum MOCK_GPU=true lokal?

RunPod kostet Geld pro GPU-Minute. Mit MOCK_GPU=true kann die gesamte Celery-Chain, JobStep-Status-Tracking und File-Saving getestet werden ohne echten API-Call. Placeholder-PNG wird ins richtige Verzeichnis geschrieben.

📖 Models erklärt

Warum UUID Primary Keys überall?

Keine sequenziellen IDs in URLs (kein /order/1/ /order/2/ — Enumeration-Angriff möglich). Besser für verteilte Systeme. id = models.UUIDField(primary_key=True, default=uuid.uuid4, editable=False)

Warum JSONField für required_export_types?

Jede Produktvariante braucht andere Dateiformate: Poster braucht CMYK TIFF, T-Shirt nur PNG 300dpi (POD). JSONField ist flexibel ohne extra Tabelle. Beispiel: ['pod', 'cmyk', 'preview']

Warum FulfillmentPartner.user = OneToOneField?

Partner-Dashboard nutzt Djangos eingebautes Auth-System. user.fulfillmentpartner gibt den Partner-Datensatz zurück. is_staff=False verhindert Admin-Zugang.

Warum GalleryImage.source_job_id kein FK?

Loses Coupling — Gallery-App weiß nichts von Jobs-App. Kein zirkulärer Import zwischen Apps. UUID-Referenz ohne DB-Constraint.

📖 GPU Pipeline erklärt

Warum .si() (immutable signatures)?

Celery-Chain übergibt normalerweise den Return-Value als erstes Argument des nächsten Tasks. .si() ignoriert den Return-Value — jeder Task lädt seinen Zustand selbst aus der DB per job_id. Verhindert unerwartete Argument-Fehler.

Warum nur UUIDs als Task-Argumente?

ORM-Objekte sind nicht JSON-serialisierbar für Redis. UUID → Task lädt Objekt selbst aus DB. Wenn zwischen Queue und Ausführung die DB geändert wurde, lädt der Task immer den aktuellen Stand.

📖 Views + Routing erklärt

Was ist eine Django View?

Funktion oder Klasse die einen HTTP-Request empfängt und HTML/JSON zurückgibt. Class-Based Views (CBV) für Standard-Listen und Details, Function-Based Views (FBV) für komplexe Logik wie AssetSelectView.

Was ist urls.py?

Mapping URL-Pattern → View. printbuddy/urls.py ist der Haupt-Router, der andere Apps via include() einbindet. path('studio/', include('studio.urls')) delegiert alle /studio/... URLs an studio/urls.py.

Warum @login_required?

Ohne Decorator wäre jede Studio-URL ohne Login zugänglich. @login_required prüft ob gültige Session existiert und leitet sonst auf LOGIN_URL weiter. Grundlage jeder Django-Sicherheit.

HTMX Polling erklärt (ADR-12)

Kein JavaScript-Framework, kein WebSocket nötig. Im Template: hx-get="/studio/job/uuid/status/" + hx-trigger="every 3s" + hx-target="#status-panel". HTMX sendet alle 3s GET-Request und ersetzt den Inhalt des Targets mit der Server-Antwort.

📖 Frontend + Templates erklärt

Django Template Language (DTL)

HTML mit Python-ähnlicher Logik. Kein JS-Framework nötig. {% for image in images %}{{ image.title }}{% endfor %} · Django escaped automatisch gegen XSS. Filter: {{ image.created_at|date:"d.m.Y" }}

static/ vs. media/

static/ = CSS, JS, Icons — in Git, per collectstatic auf VPS gesammelt, von Nginx direkt ausgeliefert (kein Django-Overhead). media/ = generierte Bilder — auf NAS (/mnt/agency_nas/), nicht in Git.

paginate_by=24

Django gibt automatisch ein page_obj in den Template-Kontext. Im Template: {% if page_obj.has_next %}<a href="?page={{ page_obj.next_page_number }}">Weiter</a>{% endif %}

Warum HTMX für Galerie-Filter? (ADR-09)

hx-get="/gallery/?cat=poster" + hx-target="#gallery-grid" lädt nur das Grid neu. Kein React, kein Vue, kein SPA nötig. Django Templates first, HTMX-Upgrade per ADR.
